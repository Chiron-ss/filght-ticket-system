# 航班票务管理系统 - 测试方案

## 1. 测试环境准备
*   **操作系统**: Windows
*   **数据库**: SQLite (`flight_system.db`，系统启动时自动生成)
*   **客户端**: FlightTicketClient
*   **服务器**: FlightTicketServer (目前逻辑集成在客户端直接访问数据库，模拟C/S架构)

## 2. 测试账号
系统启动时会自动生成测试数据，但用户需要手动注册。

*   **普通用户**:
    *   注册任意用户名（如 `user1`），密码（如 `123456`）。
*   **管理员用户**:
    *   目前系统没有内置管理员注册界面。
    *   **测试方法**: 注册一个用户名为 `admin` 的用户，然后在数据库中手动将其 `is_admin` 字段设为 `1`，或者在代码中临时修改注册逻辑赋予权限。
    *   *(注：本次更新已在数据库层支持 `is_admin` 字段，但未提供前端直接注册管理员的入口，通常管理员由后台添加)*

## 3. 功能测试用例

### 3.1 普通用户端测试

| ID | 测试功能 | 操作步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| U01 | **用户注册** | 1. 输入用户名 `test01`，密码 `123456`，电话 `13800000000`。<br>2. 点击“注册”。 | 弹出“注册成功”提示。 |
| U02 | **用户登录** | 1. 输入刚注册的账号密码。<br>2. 点击“登录”。 | 登录成功，跳转至“航班查询”页面，右上角显示“欢迎回来”。 |
| U03 | **航班查询** | 1. 出发地输入“北京”，目的地输入“上海”。<br>2. 点击“查询”。 | 列表中显示所有北京到上海的航班。 |
| U04 | **快速预订** | 1. 选中一个航班。<br>2. 点击“快速预订”。<br>3. 选择“经济舱”。<br>4. 确认支付。 | 弹出“预订成功”提示，自动跳转到订单列表。 |
| U05 | **查看订单** | 1. 切换到“我的订单”标签页。<br>2. 查看“进行中订单”列表。 | 刚预订的订单显示在列表中，状态为“已预订”。 |
| U06 | **电子登机牌** | 1. 在“进行中订单”列表中**双击**该订单。 | 弹出“电子登机牌”窗口，显示航班号、乘客姓名、二维码等信息。 |
| U07 | **取消订单** | 1. 选中该订单。<br>2. 点击“取消订单”。 | 提示取消成功，订单从“进行中”移动到“历史订单”列表，状态变为“已取消”。 |
| U08 | **行程提醒** | *(需修改系统时间或等待)*<br>当航班起飞前2小时内。 | 系统弹出温馨提示：“您的航班即将起飞”。 |

### 3.2 管理员端测试

| ID | 测试功能 | 操作步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| A01 | **管理员登录** | 1. 使用管理员账号登录（需预先在数据库设置权限）。 | 登录成功，界面出现“管理员后台”标签页。 |

## 4. 数据库扩充验证
*   **验证方法**: 在查询界面不输入任何条件，直接点击“查询”。
*   **预期**: 列表中应出现大量航班（约30天 x 每天多条航线），覆盖北京、上海、广州、成都、深圳、西安、昆明等地。

## 5. 异常测试
*   **重复注册**: 使用相同用户名再次注册 -> 提示“注册失败”。
*   **错误登录**: 输入错误密码 -> 提示“用户名或密码错误”。
*   **重复预订**: (可选) 同一航班重复预订 -> 系统应允许或提示（视具体逻辑而定）。
*   **余票不足**: 预订一个余票为0的航班 -> 提示“预订失败”。
