# 航班票务管理系统 - 功能增强与美化计划

## 1. 总体目标
参考主流OTA（在线旅游平台）应用（如携程），对现有航班票务管理系统进行UI美化和功能扩充。保持核心功能简洁易用，重点增强用户体验和系统实用性。

## 2. UI 美化 (UI Beautification)
目标：摆脱朴素的默认界面，打造现代、清爽的视觉风格。

*   **配色方案**：采用航空蓝（#0078D7）作为主色调，搭配白色背景和浅灰色辅助色，营造专业、舒适的氛围。
*   **布局优化**：
    *   **主页**：设计为仪表盘风格，顶部为导航栏（查询、我的订单、个人中心），中部为核心功能区（航班查询卡片）。
    *   **列表页**：航班列表采用卡片式设计，清晰展示起降时间、价格、航空公司Logo（可用文字代替）和剩余座位。
    *   **详情页**：弹窗或侧边栏展示航班详情。
*   **样式表 (QSS)**：编写统一的 QSS 样式表，美化按钮（圆角、悬停效果）、输入框、表格和标签。
*   **图标**：引入简单的图标（如飞机起飞、降落、日历、用户头像等）增强视觉识别度。

## 3. 功能扩充 (Functional Enhancements)

### 3.1 扩充测试数据库 (Database Expansion)
*   **数据多样性**：增加更多国内热门航线（如北京-上海、广州-成都等）。
*   **时间跨度**：生成覆盖未来30天的航班数据，包含不同时段（早班机、晚班机）。
*   **舱位信息**：确保包含经济舱和公务舱的差异化价格和座位数。

### 3.2 管理端实现 (Admin Implementation)
*   **管理员登录**：增加管理员角色或独立的管理员登录入口。
*   **航班管理**：
    *   **添加航班**：管理员可手动录入新航班信息。
    *   **取消航班**：管理员可取消异常航班。
    *   **查看统计**：查看当前系统的总订单数、总用户数等概览。

### 3.3 预定票务后实时显示 (Real-time Ticket Display)
*   **电子登机牌**：预订成功后，生成模拟的“电子登机牌”界面，显示二维码（模拟）、座位号、登机口（随机生成）、航班号、乘客姓名。
*   **订单状态流转**：订单状态细分为：`已出票`、`待出行`、`使用中`、`已完成`、`已取消`。

### 3.4 飞机起飞前提醒 (Pre-flight Reminder)
*   **模拟通知**：在主界面增加“消息中心”或“通知栏”。
*   **逻辑**：系统启动时或定时检查当前用户订单，若距离起飞时间小于24小时/2小时，弹出温馨提示：“您的航班 [航班号] 即将起飞，请注意安排行程。”

### 3.5 行程中展示预计到达信息 (In-flight Info)
*   **行程动态**：在“我的行程”页面，对于正在进行中的航班（当前时间介于起飞和到达时间之间），显示进度条。
*   **信息展示**：显示“预计到达时间”、“剩余飞行时间”。

### 3.6 行程后归档 (Post-flight Archiving)
*   **历史订单**：航班到达时间已过的订单，自动归档到“历史订单”标签页。
*   **评价入口**：(可选) 历史订单增加一个“评价”按钮，点击可简单的打分（模拟）。

## 4. 技术实现路径
1.  **数据库更新**：修改 `DatabaseManager`，增加管理员表（或字段），扩充 `addTestFlights` 逻辑。
2.  **后端逻辑**：在 `DatabaseManager` 中增加查询特定状态订单、更新订单状态的接口。
3.  **前端重构**：
    *   应用 QSS 样式。
    *   重构 `MainWindow`，使用 `QStackedWidget` 实现多页面切换（首页、搜索结果、订单中心、管理端）。
    *   实现定时器 (`QTimer`) 刷新状态和检查提醒。

## 5. 进度安排
*   **第一步**：设计并应用基础 QSS 样式，美化现有界面。
*   **第二步**：扩充数据库生成逻辑，生成丰富测试数据。
*   **第三步**：实现订单生命周期管理（提醒、行程中、归档）。
*   **第四步**：开发简易管理端功能。
